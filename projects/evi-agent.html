Evi-Agent

Agentic Evidence Retrieval System

A policy-driven, memory-aware retrieval agent for scientific literature.

View Repository

Overview

Evi-Agent is a minimal yet architecturally explicit agentic system for evidence-grounded question answering over scientific literature (e.g., PubMed abstracts). Unlike traditional RAG pipelines, this system implements a stateful agent with explicit control flow, hybrid memory tiers, deterministic stopping policies, and transparent decision logic.

Core Design Principles

  • Separation of Policy and Mechanism
  • State-driven execution via a centralized AgentState
  • Three-tier hybrid memory architecture
  • Deterministic stopping with bounded control flow
  • Safe external tool use (PubMed API)

System Architecture

The system is implemented using LangGraph with explicit node transitions:

  • Tier 1: Query Cache (LRU + TTL controlled)
  • Tier 2: Semantic Chunk Store (ChromaDB vector retrieval)
  • Tier 3: PubMed ingestion with safe network handling
  • Evidence scoring with similarity + recency + diversity bonus
  • Policy-based STOP vs FETCH_MORE decision node

Hybrid Memory Design

Tier 1 – Query Cache

Stores solved queries with payload-based reuse. Enforces TTL and LRU eviction. Policy-free by design.

Tier 2 – Chunk Store

Stores document chunks for fine-grained semantic retrieval. Enforces per-document diversity constraints.

Conversational Memory

Summary-only rolling memory updated periodically via LLM compression.

Retrieval & Scoring Logic

  • Cosine similarity over MiniLM embeddings
  • Recency decay with exponential half-life
  • Diversity bonus across independent sources
  • Confidence gating via minimum distinct documents
  • Stagnation detection for early termination

Tech Stack

Core

  • Python
  • LangGraph
  • ChromaDB
  • Sentence-Transformers (MiniLM)

Models

  • FLAN-T5 (local inference)
  • MiniLM embeddings

Why This Project Matters

Evi-Agent is not just a RAG implementation — it is an exploration of agent architecture, memory design, and policy-driven control. It demonstrates explicit reasoning about when to act, when to stop, and how to reuse prior computation safely — foundational capabilities for scalable autonomous AI systems.
