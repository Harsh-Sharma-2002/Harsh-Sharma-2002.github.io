Overview

RepoScope is a repository-aware code intelligence backend that applies retrieval-augmented generation (RAG) to explain and review code with full cross-file awareness. Instead of naive prompt stuffing, RepoScope performs structured repository indexing, semantic chunking, bounded vector retrieval, and deterministic context expansion before invoking an LLM.

Core Capabilities

  • Full repository semantic indexing
  • Cross-file contextual code explanations
  • Structured file-level code review
  • Repository-wide architectural assessment
  • Pluggable multi-provider LLM integration (BYOK)

System Architecture

RepoScope follows a layered backend architecture:

  • FastAPI API Layer
  • Service Layer (Retrieval, Prompting, LLM Dispatch)
  • Vector DB Layer (ChromaDB persistent collections)
  • Pluggable LLM Provider Abstraction

Retrieval Pipeline

  1. Clone repository at HEAD (branch-aware)
  2. Semantic, boundary-aware chunking of source files
  3. L2-normalized vector embedding per chunk
  4. Deterministic vector search with file-local filtering
  5. Window expansion around anchor chunks
  6. Prompt assembly with structured output constraints

Design Guarantees

  • No same-file redundancy during retrieval
  • Strict file-local window expansion (never cross-file bleed)
  • Deterministic, bounded context assembly
  • Structured JSON output for reviews
  • Explicit provider selection (no silent fallback)

Tech Stack

Backend

  • Python
  • FastAPI
  • Pydantic
  • Uvicorn

Retrieval & ML

  • ChromaDB (Persistent Vector Store)
  • Sentence-Transformers (MiniLM)
  • Pluggable Embedding Providers (OpenAI, Gemini, Claude, Local)
  • Multi-Provider LLM Dispatch

Why This Project Matters

RepoScope demonstrates production-aware RAG design for code intelligence. It explicitly addresses real-world constraints such as:

  • Repository scale and deterministic indexing
  • Context explosion under LLM token limits
  • Multi-provider LLM infrastructure constraints
  • Machine-readable structured outputs for automation

The system prioritizes retrieval correctness, architectural clarity, and explicit control over flashy demos — making it suitable as a foundation for production-grade code review tooling.
